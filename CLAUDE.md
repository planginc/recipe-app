# CLAUDE.md — Recipe App

## What this project is
A personal recipe management app built with React + Vite. Users save recipes (via URL import or manual entry), organize them by folder and dietary tags, track freezer inventory, and get AI-powered dinner suggestions. Designed for sharing with the Autonomy community — each person runs their own isolated instance.

## Tech stack
- **Frontend:** React 19, Vite 7, Tailwind CSS 4, React Router 7
- **Backend:** Supabase (PostgreSQL + Edge Functions) OR Convex — user picks via env var
- **AI:** Claude API (claude-sonnet-4-20250514) via Supabase Edge Functions or Convex HTTP actions
- **Hosting:** Netlify (auto-deploys from GitHub main)
- **No SSR, no build-time data fetching** — pure client-side SPA

## Dual-backend architecture

`VITE_DB_BACKEND=supabase` or `VITE_DB_BACKEND=convex` controls which backend is used.

The abstraction layer lives in `src/hooks/`:
- `useRecipes.js` — recipes (CRUD)
- `useFreezerItems.js` — freezer inventory (CRUD)
- `useInspirationVideos.js` — YouTube inspiration videos (CRUD + step capture)

Each hook selects its implementation at module load time from `src/hooks/backends/`:
- `*Supabase.js` — uses `@supabase/supabase-js` directly
- `*Convex.js` — uses `convex/react` hooks (`useQuery`, `useMutation`, `anyApi`)

`src/lib/config.js` exports `BACKEND`, `USER_TELEGRAM_ID`, `AI_CHAT_URL`, `AUTO_TAG_URL`.

## Convex setup (for Convex backend)

The `convex/` directory contains the recipe app's own Convex functions. Community members on Convex:
1. Run `npx convex dev` in this repo to create their own Convex deployment
2. Set `VITE_CONVEX_URL` to their deployment URL
3. Set `ANTHROPIC_API_KEY` in Convex env: `npx convex env set ANTHROPIC_API_KEY sk-...`
4. HTTP actions for AI features live at `<deployment>.convex.site/recipeAiSuggest` and `/autoTagRecipe`

## Supabase setup (for Supabase backend)

1. Create a Supabase project
2. Run migration: `supabase/migrations/01_initial_schema.sql`
3. Deploy edge functions: `supabase functions deploy recipe-ai-suggest --no-verify-jwt`
4. Set `ANTHROPIC_API_KEY` secret in Supabase dashboard

## Rules specific to this project
- Never hardcode Pam's Supabase project ID or Telegram ID in any file. Everything goes through env vars.
- All user data is scoped by `VITE_USER_TELEGRAM_ID` in Supabase. In Convex, data is isolated by deployment.
- `USER_TELEGRAM_ID` is used in all DB writes — if it's empty, data will be unqueryable.
- Convex functions use camelCase field names. The Convex hook adapters normalize to snake_case for the components.
- The `supabase.functions.invoke('polish-recipe', ...)` call in StepCaptureModal is Supabase-only. On Convex, it falls back gracefully.
- Netlify deploy: run `npm run build` locally first to catch errors before pushing.

## Things to avoid
- Do not use `require()` in hook files — Vite ESM doesn't support it cleanly.
- Do not add `anyApi` calls for functions that don't exist in `convex/` yet.
- Do not remove the `BACKEND === 'convex' ? useXxxConvex : useXxxSupabase` pattern — it's intentional for hook selection.
- The `convex/_generated/` directory is auto-generated by `npx convex dev` and must not be committed.
